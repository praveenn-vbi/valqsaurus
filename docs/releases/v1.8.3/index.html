Writeback#

We've also added a lot of improvements to the writeback feature which is available on valQ Collaborate. Writeback allows you to post the scenario,series data from valQ to your backend systems.

  • This feature is available in the Simulate > Navigation Panel section on the runtime.

Writeback feature accessible from the Navigation Panel

  • The settings are available under the Model > General > Writeback section.

Writeback settings

New Writeback features:

  1. Writeback data format (V2)

    The new format will include the below details in the writeback JSON

    1. Logged in valQ user details

    2. "result" property will contain the list of scenario/series which were selected for writeback

    3. "isPrimary/isComparison" properties will be available on series. This can be used during processing to check for base and comparison series.

    4. Node level comments will also be included in the writeback

    Writeback Format

  2. Allow writeback of Simulated Nodes:

    The section Exporting Nodes, has an option - "Only simulation affected nodes".

    By selecting the above option, you will restrict the export of just the Nodes affected in the Simulation.
