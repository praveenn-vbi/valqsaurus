Analyze Tab

The Analyze tab allows you to run advanced analyses on the data. Currently, you can perform variance analysis from this tab. Expect to see additional items in the future such as sensitivity analysis, attribution analysis and more.

#Variance Analysis


Variance Analysis helps analyze variances against a benchmark for specific periods for all the KPIs in the model.

By default, you will see the below screen upon clicking on Variance Analysis underthe Analyze tab.



The Analyze tab consists of two panels:

On top of the page you will find a panel containing the features that help to configure the analysis report as per your needs.



The other panel shows the Variance Analysis report prepared based on the model that you have built and it's configuration.



Now let's take a look at the features of Variance Analysis.

#1. Set Base Series and Compared Series

Variance Analysis helps compare the values of Base Series with that of the Compared Series for each of the node in the model and calculate variances for the nodes.

You can set the Base Series and Compared Series from the model that you want to compare here.

#2. Set a Scenario as Base Series

In the Analyze tab, you can implement the scenario that you have created in the Simulate tab as Base Series and compare it with Compared Series to perform Variance Analysis. This can be done by simply choosing the scenario that you want from the drop down and clicking on Apply scenario option.

#3. Choose time frame to perform analysis

The Period dropdown allows you to specify time frame for which you would like to run the analysis. Here, you can select multiple time frames like Full year, Quarter, Year-to-date, etc.

#4. Set Base period and Compared period to calculate variances


There are few properties in this section

Base Period: By default, this will show the first period in the ‘Simulation Period’ from the ‘Simulate’ tab. This is the period that will show up whenever you would like to analyze variance for the ‘current period’.

Comparison Period: By default, you can analyze performance vs. the benchmark for the same period.
E.g., You can change the period to July Budget vs. July Forecast by just clicking on the drop down and choosing Jul.

However, if you would like to compare July Budget vs. Jan Forecast, select ‘Custom' option on the drop down and set the Base period and Compared period as below.

With 'Compare Time Range' option, you can add another custom time frame in addition to the time frames already selected in (2). Let's add to the report Jan-Apr analysis as below.

#5. Show Absolute & % Variance

In this section you may choose which variances are important to you for analysis. Selecting these options adds them to the table. The positive variance items are shown in green and the negative ones in red.

#6. Show Hidden Nodes

Here, you have the option to display the nodes that were hidden while building the model under the Model tab.

Note
Upon enabling show hidden node, the node stays hidden on the other tabs.

#7. Show/ Hide Unit column

The unit column is seen by default on the Analyze tab. To hid the column, click on show unit to disable the feature.

#8. Apply Filter to report

Use the Filter option to analyze critical influencers. Two filters are available:

Filter by Parent: This allows you to analyze variance for a specific branch within the model. For example, you may be interested in analyzing variance for a particular location or product.

Filter by Variance: This allows you to identify top ‘n’ analysis for categories showing the most variances for particular period choosen from the drop down.

Note
You have the option to select whether you want to display only positive variances or only negative variances or both.


In the below image, filter By Variance has been applied to filter top 10 positive variances of the periodic report. The Value Bar displays the filter that has been applied. To remove the filter simply click on the 'x' next to the filter.

#9. Search for a node

You can search for a particular node by using the search bar as highlighted below. This feature can be used to identify a node when a model has large number of nodes.

#10. Hide the features panel

Click on '^' to hid the features panel of the Analyze tab. The screen will be seen as below after the panel is hidden.
