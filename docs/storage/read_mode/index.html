Read-only Mode

Those users who are not part of the ValQ workspace are presented with Read-only mode capabilities which allows them to view the model and perform run-time operations without enforcing login. This feature is beneficial for embedding Power BI reports.

Note
Read-only mode is not available in Power BI Desktop

#Reading View

Read-only mode is available only in Power BI Service as Reading View which is it's default mode.


Let's take a look at the ValQ tabs on Reading View of Power BI Service.

#1. Model

Users need to be part of the ValQ workspace to access Model tab.

#2. Plan

Planning Editor cannot be used on Read-only view.


However, in case of existing planning series, the users can view the plan. Notice that the Plan features are disabled.

#3. Simulate

On the Simulate tab, the users can view the model, perform simulations, switch to tabular view, create, compare, and export scenarios.


However, the Scenarios comes with a warning message to the Reading View users as shown below. The scenarios created under read mode gets deleted on clicking Reset to Default icon on Power BI Service.


Note
The user is required to log in to ValQ workspace to save the scenarios created under read mode.

#4. Analyze

Variance Analysis is fully functional on Reading View.


Reading Viewe users can make use of ValQ's Attribution Analysis.

#Edit Report

Users who are part of the workspace can click the Edit Report option on Power BI Service and then login to workspace to get the complete access and use ValQ.
