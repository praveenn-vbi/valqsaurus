Understanding Nodes in ValQ

With ValQ, you can build a model of your business process and view it in the form of a tree as well as table. Toggle between the below icons present on top left corner of the canvas to change the view.

#Tree View

The tree structure is presented in horizontal format as a visual. Just like a tree, ValQ model has a stem on top called top node and the top node then branches out to others nodes.


The model is segmented into levels starting from level 1 with top node. The nodes share a parent-child relationship at each level for example, 'East' node is the parent node while 'Consumer' node is a child node, 'Consumer' node is the parent node while 'Chris Morris' node is a child node.


Click on '>' to expand and '<' collapse levels.


You can also use the 'Go to level ^' option present on bottom of the canvas to navigate between levels of the tree.


Now let's take a look at what makes up a node.


Each node represents a value driver, KPI or category and it's several performance metrics. Here's how to interpret a node:

ComponentDescription
1Node nameName of the value driver or KPI
2Conditional FormattingColor band indicating the variance
3Value TrendA sparkline line graph showing the trend
4Baseline ValueNode value of the baseline series
5Variance for all periodsFull year variance of the node
6Variance for current periodcurrent month (usually first period in the series) month variance of the node
7Aboslute VarianceVariance shown in absolute value
8Percentage VarianceVariance shown in percentage

Note
Variance is calculated as difference between Baseline series and Comparison series

Baseline series is the primary series and Comparison series is the benchmark against which the Baseline is compared. Here Sales Budget is the Baseline series and Sales Forecast is the Comparison series.
