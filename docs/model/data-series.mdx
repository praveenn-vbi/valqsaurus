The **Model > Data Series** tab helps you to configure the data series settings for the model created in ValQ. The options available are:

1. [Data Series Manager](#data-series-manager)
    1. [Periods](#periods)
    1. [Data Series](#data-series)
1. [Time Aggregation Labels](#time-aggregation-labels)
1. [Data Sorting](#data-sorting)
1. [Linked Model Mapping](#linked-model-mapping)

## Data Series Manager

Data Series Manager helps you to configure the <Link to={useBaseUrl('model/data-series#11-periods')}>Periods</Link> and <Link to={useBaseUrl('model/data-series#12-data-series')}>Data Series</Link> for the model.

    <img alt="New Tab" src={useBaseUrl("/doc-images/model/ds2.png")} />
  </Zoom>
  <div class="center"><p>Data Series Manager setting</p></div>

### Periods

**Period(s) per data series**: Enter the number of periods in the data here. <br />
**Period Label(s)**: Enter the period labels here with a comma separating each label.<br />
The changes made here to the Period Labels will be reflected in all 4 tabs on ValQ - Model,
Plan, Simulate and Analyze including the PopUp screen.

### Data Series

<div style={{ textAlign: "justify" }}>
  First, toggle Include comparison data series to add a comparison data series
  when needed. In case there are two data series to the model, you can keep one
  as Baseline data series and the other as Comparison data series. The data
  series can be actuals and budget, multiple years of data, etc. The names of
  the series are Baseline and Comparison by default, you can double click on the
  name or click on the pencil icon beside the name to customize.
</div> <br />

<div style={{ textAlign: "justify" }}>
  For our example, we have the data source having periods from Jan to Dec,
  Budget and Forecast values. The Budget values will be the Comparison values
  and the Forecast values will be the Baseline values. Follow the below steps
  for configuring the values in the Data Series Manager.
</div> <br />

1.  Set the property Period(s) per data series to the value 5.
2.  Configure the Period Labels as Jan, Feb, Mar, Apr, May (see Figure
    above).
3.  Enable the property Include comparison data series. You can view
    both the Baseline and Comparison values.
4.  When the property Include comparison data series is disabled, then
    you will be able to view only the Baseline Series.
5.  When you click the Baseline label, you will be able to view the
    Baseline values as Forecast values as shown below. Now the label
    has been changed to Sales Forecast.

</div>

6.  When you click the Comparison label, you will be able to view the
    Comparison values as the Budget values as shown below. Now the label
    has been changed to Sales Budget.

## Linked Model Mapping

Use this **Linked Model Mapping** when you want to map the series/scenarios of your model with a sub-model(or node) that you have linked.
Mapping the series/scenarios of the sub model helps in keeping your model in sync with the changes in the sub-models.

:::important
This feature is only enabled when you have linked a node with another sub-model. If not there is a warning pop-up message as shown below:

<div class="center">
  <Zoom>
    <img alt="Linked Series Mapping" src={useBaseUrl("/doc-images/model/linked-model-warning.png")} />
  </Zoom>
  <p>Link a node with a sub-model message</p>
</div>
:::

* Link a node to an entire sub-model or a node from the sub-model. [ Refer the <Link to={useBaseUrl('storage/model-merge')}>Model Merge</Link> section ]
* Select **Model > Data Series > Linked Model Mapping**.

<div style={{ textAlign: "center" }}>
  <Zoom>
    <img alt="Linked Model Mapping" src={useBaseUrl("/doc-images/model/linked-model-map.png")} width="300" />
  </Zoom>
</div>

* Select the **Map Series** tab and map the respective series as indicated below:

<div class="center">
  <Zoom>
    <img alt="Map Series" src={useBaseUrl("/doc-images/model/map-series.png")} />
  </Zoom>
  <p>Series mapping between the linked node and the sub-models</p>
</div>

* Select the **Map Scenarios** tab and map the respective scenarios(only shared):

<div class="center">
  <Zoom>
    <img alt="Map Scenarios" src={useBaseUrl("/doc-images/model/map-scenarios.png")} />
  </Zoom>
  <p>Scenarios mapping between the linked node and the sub-models</p>
</div>

Now to sync up the series/scenarios in the consolidated model refer the <Link to={useBaseUrl('storage/model-merge#final-result-in-the-consolidated-model')}>Final result in the consolidated model</Link> section for further details.
